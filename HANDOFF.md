## BUILD PIPELINE (build_scene.py)
Generates two files from config.json:
1. scene_files/volumes/rgbgrid.pbrt — MakeNamedMedium "rgb_vol" block
2. scene_files/scene.pbrt — complete pbrt-v4 scene

### Script structure (7 sections):
1. Utility functions (fmt_floats)
2. wavelength_to_rgb — 380-700nm to RGB
3. compute_rgbgrid — builds sigma_s/sigma_a/le_flat voxel arrays
4. write_medium — generates rgbgrid.pbrt (skipped if grid.enabled=false)
5. Scene block writers — one function per pbrt concept
6. write_scene — assembles scene.pbrt in correct pbrt order
7. main — entry point, accepts config path as argument

### Correct pbrt-v4 scene order (critical):
```
MakeNamedMedium "fog"       <- fog declaration (if enabled)
MediumInterface "" "fog"    <- camera born inside fog (if enabled)
LookAt ...
Camera ...
Sampler ...
Integrator ...
Film ...
WorldBegin
    Include "scene_files/volumes/rgbgrid.pbrt"   <- if grid.enabled
    Include "scene_files/tree.pbrt"               <- if tree.enabled
    LightSource ...
    AttributeBegin
        MediumInterface "rgb_vol" ""   <- before Material
        Material "interface"
        Translate / Rotate ...
        Shape ...
    AttributeEnd
```

## SPACE COLONIZATION (space_col.py)
Standalone module at repo root. Called by render_pipeline.sh when tree.enabled=true.
Generates scene_files/tree.pbrt — cylinders + joint spheres.

### Script structure (7 sections):
1. Imports (os, math, random, json, numpy, scipy.spatial.KDTree)
2. Leaf3D — attraction point with Poisson disk placement
3. Branch3D — branch node with parent pointer, Murray radius
4. Tree3D — algorithm driver
   - _generate_leaves() — Poisson disk (dart throwing) sampling in ellipsoid
   - _grow_trunk() — grows upward until within max_dist of canopy
   - grow() — main space colonization loop with stall detection
   - _compute_murray_radii() — basipetal Murray's law r^n = sum(child r^n)
   - get_cylinders() — returns (parent_pos, child_pos, radius) list
   - get_joints() — returns (pos, radius) list with joint_radius_cap
5. write_tree() — emits pbrt AttributeBegin/End blocks for cylinders and spheres
6. run() — entry point called from render_pipeline.sh
7. main — CLI entry point: python3 space_col.py <config.json>

### Key algorithm parameters (Runions notation):
- D — segment length (growth_dist)
- dk_multiplier — kill distance as multiple of D (min_dist = dk_multiplier * D)
- di_multiplier — radius of influence as multiple of D (max_dist = di_multiplier * D)
- birth_dist — minimum spacing between attraction points (Poisson disk)

### Runions Figure 3 parameter sets:
- Form a: N=12000, dk=2D, di=inf  — dense full crown
- Form b: N=1500,  dk=2D, di=inf  — open visible branches
- Form c: N=12000, dk=20D, di=inf — sweeping curved branches
- Form d: N=375,   dk=20D, di=inf — very sparse dramatic

### Murray's law radius:
- r^n = sum(child_r^n), n=2, working basipetally from tips
- base_radius — tip node radius (finest branches)
- trunk_radius — cap on maximum radius (prevents absurd trunk size)
- joint_radius_cap — cap on joint sphere radius (prevents trunk bumps)
- joint_radius_multiplier — joint sphere slightly larger than cylinder

## KEY pbrt-v4 LESSONS LEARNED
- Include for rgbgrid medium goes INSIDE WorldBegin, not before it
- MediumInterface must come BEFORE Material in AttributeBegin block
- Fog MakeNamedMedium and MediumInterface must come BEFORE LookAt/Camera
- rgbgrid bounding box (p0/p1) is always world-space axis-aligned
- Box container uses trianglemesh (NOT bilinearmesh) — inward-facing winding
- Cylinder shape has no end caps — use spheres at joints
- pbrt native box primitive does not exist — use trianglemesh with 8 vertices
- Distant light is correct for sunlight simulation (parallel rays, from/to)
- rgbgrid supports volumetric emission via "rgb Le" and "float Lescale"
- Emissive volumes need high pixel_samples (512+) to resolve cleanly
- Grid enabled flag skips both rgbgrid.pbrt generation and Include in scene

## NEXT SESSION WORK — FOLIAGE SYSTEM
Implement foliage grounded in Runions papers and Algorithmic Botany.
Four phases:

### Phase 1 — Parallel Transport Frame
- Compute consistent local coordinate system at every branch node
- Prevents leaf twist as branches curve through space
- Math: iterative frame propagation from root to tips
- Reference: Runions tree paper, Algorithmic Botany Ch 1

### Phase 2 — Phyllotaxis Placement (Algorithmic Botany Ch 4)
- Cylindrical model, equation 4.2: phi = n * 137.5 degrees, H = h * n
- Walk each branch path root to tip
- At each node rotate by golden angle 137.5 around branch axis
- Parameters: h (internode distance), starting angle
- Config: "phyllotaxis": { "enabled": true, "h": 0.3, "start_angle": 0 }

### Phase 3 — Leaf Shape via Venation Algorithm
- Port space colonization to 2D within parametric leaf boundary
- Boundary defined as ellipse or parametric curve in config
- Open venation pattern (tree-like, Runions venation paper Section 3)
- Produces vein network as 2D graph
- Murray's law for vein width (venation paper Section 3.5)

### Phase 4 — Leaf Mesh + pbrt Output
- Veins as thin trianglemesh ribbons
- Blade as triangulated surface
- ObjectBegin/ObjectInstance for efficient instancing
- Orient using Phase 1 frame + Phase 2 phyllotaxis angle
